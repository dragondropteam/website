Getting Started With DragonDrop

These tutorials will teach you all of the basics about how to use DragonDrop and it's different environments.

DragonDrop

This set of steps will help you become familiar with DragonDrop's block based environment!

A Visual Phaser project will be used in this example, but the concepts apply to all block based DragonDrop projects.

This is what the basic DragonDrop environment looks like!

1:
This is where all of the usable blocks can be found! Click on a category to find relevant blocks.

Check out our documentation to learn about how all of the different blocks work!

2:
This is where all of the block programming takes place! All blocks should go inside of a surrounding parent block. In Phaser, this means that all blocks should go in the "run phaser" block, unless they are in functions or classes.

3:
This is the live Javascript output. For every block you add or delete, it will immediately edit the equivalent Javascript accordingly.

You cannot type in this window, but you can copy it's contents to use in the Javascript version of Phaser in DragonDrop.

Visual Phaser

Get started by creating a Visual Phaser project.

Click on Create Project, choose Visual Phaser Game Engine.

Make sure you give it a name that makes sense and save it in a place you can easily find!
